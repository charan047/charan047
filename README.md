### 👋 About

Senior software engineer focused on **backend architecture, distributed systems, and production AI/ML**. I design services that hold up under load, own systems end-to-end from API contract to deploy, and treat AI coding agents as part of the toolchain — orchestrated with the same rigor as any other production system.

- 🏗️ I work at the **system level**: API design, data modeling, scaling, reliability

- 🔧 I value clean abstractions, observable systems, and code that's easy to delete

### 🚀 AI-Augmented Engineering

The 2026 SDLC moved from autocomplete to **delegated, agent-driven development**. I work the way modern teams ship — using agents for leverage while keeping architecture, security, and correctness as my responsibility.

| Practice | How I work |
|---|---|
| **Agentic coding** | Drive [Claude Code](https://docs.claude.com/en/docs/claude-code/overview) & CLI agents for multi-step refactors, migrations, and test generation — plan → execute → review loops |
| **Context engineering** | `CLAUDE.md` / agent instruction files, scoped context, and repo conventions so agents produce consistent, reviewable output |
| **MCP integration** | Build & wire [Model Context Protocol](https://modelcontextprotocol.io) servers so agents safely reach databases, APIs, and internal tools |
| **Guardrails & oversight** | Sandboxing, scoped permissions, human-in-the-loop approval for high-risk changes, PR-based review of all agent output |
| **Rapid prototyping** | Fast prompt-driven iteration ("vibe coding") for spikes & demos — then harden into production-grade, tested code |

> I treat agents like capable junior engineers: useful for leverage, never trusted blindly. Every line ships through tests and review.
